## 修复：讯飞数字人无法正常显示（黑屏）

**场景**：日常开发 — 修复 Bug

**根因**：`frontend/tourist-app/src/components/DigitalHuman/XunfeiAvatar.vue` 创建
`AvatarPlatform` 实例时未传入 `useInlinePlayer: true`。

讯飞 avatar-sdk-web 3.2.3.1002 内部逻辑：
- `new AvatarPlatform(props)` 构造函数仅在 `props.useInlinePlayer === true` 时
  调用 `createPlayer()`。
- `start({ wrapper })` 内部的 `No(streamInfo, wrapper)` 函数检查 `useInlinePlayer`，
  为真才会执行 `createPlayer()` → `player.container = wrapper` → `player.playStream()`。
- 不传该参数时（默认 falsy），SDK 走 else 分支直接忽略，不创建 player、不挂载
  wrapper、不播放视频流 → 数字人区域始终黑屏，且无任何错误日志。

**修复**：
- `new AvatarPlatform()` → `new AvatarPlatform({ useInlinePlayer: true })`
- `bindEvents` 增加 `SDKEvents.connected` / `stream_start` 日志，便于排查连接状态
- 增加 `PlayerEvents.error` 监听 + `playNotAllowed` 时输出告警
- `resume()` 调用改为可选链 `avatar?.player?.resume?.()` 提升健壮性

**验证范围**：
- 改动类型安全（匹配 `IAvatarPlatformProps` 类型定义）
- 后端 `/api/v1/avatar/xunfei/signed-url` 路由已注册（main.py），.env 六项配置齐全
- `DigitalHuman.vue` 三模式编排 + `ChatView.vue` 降级链路逻辑正确，无需改动

**关键文件**：
- `frontend/tourist-app/src/components/DigitalHuman/XunfeiAvatar.vue`（核心修复）
- `frontend/tourist-app/public/sdk/avatar-sdk-web_3.2.3.1002/esm/index.js`（SDK 源码，
  用于定位 useInlinePlayer 的作用）
- `backend/app/api/avatar.py`（签名接口，无需改动）

**注**：前端依赖存在 peer 冲突（typescript@5.5 vs @vue/tsconfig 要求 >=5.8），
需用 `npm install --legacy-peer-deps` 安装。

## 全面美化前端界面 + XunfeiAvatar wrapper 显示修复

**追加场景**：用户反馈数字人仍然黑屏（start 成功但视频画面空），要求全面美化所有界面。

### 真正根因：wrapper 渲染层问题

`useInlinePlayer: true` 修复后，start 走通了（"讯飞"徽章亮起），但视频画面仍空。
SDK 把 `<video>` 元素插到 `.xunfei-wrapper` 内部后，因为：
1. wrapper 在 flex 容器中 `height: 100%` 行为不可靠（flex item 默认收缩）
2. SDK 内部创建的 video 元素没有显式 width/height，元素存在但尺寸为 0

### 修复（XunfeiAvatar.vue）

- `.xunfei-avatar` 加 `flex: 1; min-height: 0` 撑满父级 .stage
- `.xunfei-wrapper` 改用 `position: absolute; inset: 0` 强制占满
- `:deep(video) / :deep(canvas)` 强制 `width/height: 100% !important; object-fit: contain`
- 增加连接状态文字 + spinner + 失败遮罩（取代原先简陋的"连接中…"）
- 徽章加 live-dot 呼吸效果
