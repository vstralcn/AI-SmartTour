- API 层保持零业务逻辑，路由函数仅做参数转发并直接 return 对应 service 函数的结果。
- 数据聚合统一通过 `async with session_scope() as session:` + `session.scalars(select(Model)).all()` 获取全量记录后在内存用 `Counter`/`defaultdict` 统计，避免复杂 SQL。
- 对外返回结构采用预定义的 Pydantic response_model（`DashboardData`、`SentimentReport`），由 service 构造 dict 自动序列化。
- 时间序列趋势统一以 `_last_seven_days(today)` 生成的过去七天为基准，缺失日期补 0，保证前端图表横轴稳定。
