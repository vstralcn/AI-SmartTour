本模块由三个文件构成应用顶层装配层与数据访问层：
- `main.py` 作为 FastAPI 应用入口，通过 `lifespan` 协程在启动时调用 `init_database()` 和 `seed_defaults()` 完成数据库初始化与种子数据填充；注册 CORS 中间件并将 `chat/recommend/knowledge/avatar/analytics/digital_human_broadcast` 等子模块的 `router` 统一挂载到 `/api/v1` 前缀下，同时暴露 `/health` 聚合 `dialogue_engine.rag.knowledge_base` 大小与 `database_is_ready()` 状态。
- `config.py` 基于 `pydantic_settings.BaseSettings` 声明全局 `settings` 单例，从 `.env` 加载数据库 URL（默认 SQLite+aiosqlite）、Redis、DashScope LLM、向量库路径、数字人服务地址以及 ASR/TTS 模型路径，为各子包提供集中配置源。
- `services/persistence.py` 是面向业务实体的异步 DAO 层，所有写操作均通过 `app.db.session_scope` 获取异步 Session 并在 `async with` 块内 `commit`；读/写 `KnowledgeDocumentRecord`、`AvatarRecord`、`ConversationSessionRecord`、`InteractionRecord` 后统一调用 `sync_rag_from_database()` 将变更同步回 `app.core.rag.rag_engine`，实现 DB ↔ RAG 的双向一致性；`seed_defaults()` 在首次启动时写入默认 Avatar 并从 `rag_engine.export_entries()` 回填知识库。
依赖方向：`main.py` → `config.py` + `services/persistence.py` + 各 `app.api.*` 路由；`persistence.py` 仅依赖 `app.db`（session/models）与 `app.core.rag`，不反向依赖 API 层。
