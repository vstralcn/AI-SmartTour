- 所有数据库写操作使用 `async with session_scope() as session:` 上下文管理器包裹，在块内显式 `await session.commit()`，避免手动管理连接生命周期。
- 对 KnowledgeDocumentRecord 与 AvatarRecord 的 save 方法采用“先 get 判断存在再 add 或逐字段更新”的 upsert 模式，并统一设置 `updated_at = datetime.now()`。
- 任何修改 KnowledgeDocumentRecord 或 AvatarRecord 的持久化函数在 commit 后都会调用 `sync_rag_from_database()`，保证数据库与内存 RAG 索引一致。
- 配置项集中在 `config.Settings` 类中并以 `settings = Settings()` 导出单例，供其他子包通过 `from app.config import settings` 直接引用。
