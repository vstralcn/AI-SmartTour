模块由三层组成：
- API 层（`app/api/avatar.py`、`app/api/digital_human_broadcast.py`）：基于 FastAPI `APIRouter` 暴露两组路由——`/admin/avatar`（管理端形象配置）与 `/avatar`（公开查询），以及 `/digital-human/broadcast*`（异步任务型播报）。路由仅做参数校验、状态码映射与响应封装，不持有业务逻辑。
- 持久化层（通过 `app.services.persistence` 间接访问）：`avatar.py` 调用 `save_avatar_record` / `activate_avatar_record` / `delete_avatar_record` / `list_avatar_records` / `get_active_avatar_record` 操作 `AvatarRecord` 模型，完成形象配置的增删改查与激活切换。
- 推理服务桥接层（`app/services/digital_human.py`）：预留 `generate_talking_head` 接口对接 MuseTalk 推理服务，当前为占位实现；实际的高清播报走 `digital_human_broadcast.py` 内嵌的 HTTP 客户端（httpx）直接轮询 `settings.digital_human_url` 下的 `/generate`、`/jobs/{id}`、`/jobs/{id}/video` 三阶段协议。

播报流程采用“提交→轮询→下载”三段式异步模式：`POST /broadcast` 创建 job 并启动后台协程 `_poll_dh` 或 `_local_simulate`，`GET /broadcast/{id}` 返回 `{status, progress, engine, has_audio}`，`GET /broadcast/{id}/video` 以 `FileResponse` 输出落盘视频。当 remote 服务不可达且系统无 ffmpeg 时返回 503，供前端降级为实时 TTS。任务状态暂存于进程内 `jobs: dict[str, dict]`，生产环境注释标注应替换为 Redis。
