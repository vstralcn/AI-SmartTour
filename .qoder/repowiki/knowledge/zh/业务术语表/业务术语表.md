### 可信 RAG
- 定义：SmartTour 项目中指代具备来源引用、置信度评分与阈值拒答能力的检索增强生成机制。当前阶段基于 FAQ 关键词匹配 + 轻量排序，目标是当证据不足时主动拒答而非编造答案，与通用 ChatGPT 式问答形成差异化卖点。
- 别名：可信检索、RAG 拒答

### 游客画像
- 定义：系统从对话中动态抽取并维护的游客偏好状态，包括兴趣标签（历史文化/自然风光/民俗体验/美食探索/摄影打卡/亲子游玩）、同行人员（儿童/老人）、行动能力（标准/低强度）、游览时长等维度，用于个性化路线推荐与讲解风格调整。
- 别名：user_profile、游客状态

### Agent 工具白名单
- 定义：导览 Agent 被允许调用的受控函数集合，当前包括 knowledge（知识检索）、route（路线规划）、profile（画像更新）、feedback（反馈记录）。模型不能直接执行外部操作，所有动作必须通过白名单工具完成，以保证安全可控。
- 别名：tool whitelist、受控工具

### grounded_answer
- 定义：当 LLM 调用失败、超时或 RAG 置信度低于阈值时，系统返回的基于本地知识库的可信兜底回答，避免产生幻觉内容。这是 SmartTour 区别于通用大模型的关键降级策略。
- 别名：可信降级回答、兜底回答

### 数字人播报
- 定义：将 LLM 生成的讲解文本通过 TTS 转换为语音，并驱动数字人形象进行口型同步播放的多模态输出流程。当前浏览器端使用 SpeechSynthesis 驱动 2D 口型状态，服务端预留 MuseTalk GPU 推理链路用于真人口型同步。
- 别名：TTS 播报、口型同步

### 离线评测集
- 定义：SmartTour 项目计划构建的 200 题人工标注问题集，用于自动化评估 RAG 准确率、Recall@5、拒答率、P95 延迟等量化指标。README 强调"未实测不宣传"，评测集是答辩时展示真实数据的必要支撑。
- 别名：评测集、标准问题集

### 服务闭环
- 定义：SmartTour 的核心产品叙事：游客扫码进入 → AI 完成导览服务 → 景区获得匿名洞察 → 运营人员补充知识/优化服务 → 问答和满意度提升 → 形成续费价值。这是商业落地部分的最小闭环描述。
- 别名：运营闭环、服务—分析—优化—再服务
